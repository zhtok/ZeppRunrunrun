# ZeppRunrunrun

ZEPP Life API 刷微信支付宝步数

## ✨ 新功能

### Token 缓存机制（2026-01-14）
- ✅ 支持 Token 缓存，减少登录频率
- ✅ AES-128 加密存储，保护账号安全
- ✅ 自动复用有效 Token，降低风控风险
- ✅ 详细错误日志，便于问题排查

## 📦 快速开始

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 配置环境变量
在 GitHub Secrets 或本地环境中配置：

| 变量名 | 说明 | 必需 | 示例 |
|--------|------|------|------|
| `CONFIG` | 配置JSON | ✅ | 见下方说明 |
| `AES_KEY` | 16字节加密密钥 | ⭐ 推荐 | `1234567890abcdef` |

### 3. CONFIG 配置示例
```json
{
  "USER": "13800138000#user@example.com",
  "PWD": "password1#password2",
  "MIN_STEP": 18000,
  "MAX_STEP": 25000,
  "SLEEP_GAP": 5
}
```

## 🔐 Token 缓存配置

**强烈推荐**配置 `AES_KEY` 以启用 Token 缓存功能：

1. 生成 16 字节密钥（16 个字符）
2. 在 GitHub Secrets 中添加 `AES_KEY`
3. 首次运行后会自动生成 `encrypted_tokens.data` 文件
4. 后续运行将自动使用缓存的 Token

详细说明请查看：[Token 缓存功能文档](docs/TOKEN_CACHE.md)

## 🖥️ Windows 本地使用方法

### 1. 安装依赖

双击运行以下任一方式安装依赖：

**方式一**：在命令行中运行
```powershell
pip install requests pytz pycryptodome
```

**方式二**：确保本机已安装 Python 3.x，直接运行 `run.bat`

### 2. 配置 config.json

打开项目文件夹中的 `config.json` 文件，修改配置：

```json
{
  "USER": "账号(手机号或邮箱)",
  "PWD": "密码",
  "MIN_STEP": 最小步数(填数字即可),
  "MAX_STEP": 最大步数(填数字即可，最好别超过5万步),
  "SLEEP_GAP": 5,
  "AES_KEY": "(可留空)",
  "RUN_MODE": "interactive",
  "RUN_START_TIME": "07:00",
  "RUN_END_TIME": "22:30",
  "RUN_INTERVAL": 3
}
```

#### 配置项说明

| 配置项 | 说明 | 必填 | 默认值 |
|--------|------|------|--------|
| `USER` | Zepp 账号（手机号或邮箱） | ✅ | - |
| `PWD` | Zepp 密码 | ✅ | - |
| `MIN_STEP` | 随机步数最小值 | ⭐ 推荐 | 18000 |
| `MAX_STEP` | 随机步数最大值 | ⭐ 推荐 | 25000 |
| `SLEEP_GAP` | 多账号执行间隔（秒） | 否 | 5 |
| `AES_KEY` | Token 缓存密钥（16字符，留空跳过） | 否 | "" |
| `RUN_MODE` | 定时任务模式：`interactive` 或 `server` | 否 | "interactive" |
| `RUN_START_TIME` | 定时任务开始时间（仅导入任务时使用） | 否 | "07:00" |
| `RUN_END_TIME` | 定时任务结束时间（仅导入任务时使用） | 否 | "22:30" |
| `RUN_INTERVAL` | 定时任务执行间隔（小时，仅导入任务时使用） | 否 | 3 |

#### 账号配置（支持多账号）

多个账号用 `#` 分隔，密码也要对应数量：
```json
"USER": "13800138000#user@example.com",
"PWD": "password1#password2"
```

#### AES_KEY 配置（可选）

- **留空 `""`**：不启用 Token 缓存，每次都重新登录
- **填写 16 个字符**：启用 Token 缓存，例如 `"1234567890abcdef"`
- 启用后会生成 `encrypted_tokens.data` 文件，下次运行自动复用 Token

### 3. 运行程序

**方式一：手动运行**
- 双击 `run.bat` 即可立即运行一次

**方式二：设置定时任务（推荐）**
1. 双击 `cron.bat`
2. 按提示选择运行模式：
   - `1` 交互式模式（仅登录时运行，适合个人电脑）
   - `2` 服务器模式（不登录也能运行，适合服务器/远程桌面）
3. 服务器模式需要输入 Windows 登录密码
4. 按提示设置时间：
   - 输入每天开始执行时间（默认 07:00）
   - 输入每天最后执行时间（默认 22:30）
   - 输入执行间隔小时数（默认 3）
5. 完成后会显示所有执行时间点
6. 之后每天会自动按设定时间运行

### 4. 定时任务说明

#### 两种运行模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **交互式模式** | 只有登录电脑时才运行 | 个人电脑、笔记本 |
| **服务器模式** | 不登录也能运行 | 服务器、远程桌面 |

- 服务器模式需要输入 Windows 登录密码，密码由 Windows 任务计划程序安全存储
- 可以在 `config.json` 中设置 `RUN_MODE` 为 `"interactive"` 或 `"server"` 来指定默认模式

#### 执行时间示例

运行导入脚本后，会自动生成以下执行时间（以默认配置为例）：
```
07:00 → 10:00 → 13:00 → 16:00 → 19:00 → 22:30
```

如果需要修改执行时间，只需：
1. 修改 `config.json` 中的 `RUN_START_TIME`、`RUN_END_TIME`、`RUN_INTERVAL`
2. 重新运行 `自动导入定时任务每日22点30分自动执行.bat`

### 5. 查看和管理定时任务

- 打开 **任务计划程序**（在开始菜单搜索"任务计划程序"）
- 找到任务名称 `步数修改`
- 可右键查看属性、删除、立即运行等操作

### 6. 常见问题

**Q：提示"加载 token 缓存失败"怎么办？**
A：旧的缓存文件格式损坏，程序会自动删除重建，直接重新运行即可。

**Q：提示修改成功但实际没改？**
A：可能是 Token 过期，请删除 `encrypted_tokens.data` 文件后重新运行。

**Q：需要管理员权限吗？**
A：导入定时任务时需要管理员权限，程序会自动请求。

**Q：如何停止定时任务？**
A：在任务计划程序中删除 `步数修改` 任务，或运行以下命令：
```powershell
schtasks /delete /tn "步数修改" /f
```

**Q：服务器模式和交互式模式有什么区别？**
A：
- 交互式模式：只有你登录电脑时才运行，适合个人电脑
- 服务器模式：不登录也能运行，适合服务器/远程桌面，需要输入 Windows 密码

**Q：服务器模式密码安全吗？**
A：密码由 Windows 任务计划程序安全存储，不会明文保存在文件中。

**Q：任务计划程序报错"操作员或系统管理员拒绝了请求"怎么办？**
A：这是 S4U 登录类型的权限问题，请使用服务器模式（输入 `2`）重新导入任务。

## ⚠️ 注意事项

1. **AES_KEY 必须是 16 字节**（16 个字符）
2. 不要将 `encrypted_tokens.data`提交到 Git
3. 保管好 AES_KEY，泄露后需立即更换
4. 未配置 AES_KEY 时，功能会自动降级为每次登录模式
